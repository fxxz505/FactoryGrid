# FactoryGrid

FactoryGrid 是一款运行在浏览器中的模块化自动工厂模拟器，使用 Vue 3、TypeScript 和 Canvas 2D 构建。玩家可以在网格画布上铺设传送带、连接机器、配置配方，并通过生产研究包、输送到研究中心来推进技术与工厂升级。

项目采用简单几何图形绘制机器和物品，不依赖外部图片素材。矿石使用圆形表示，工业产品使用三角形表示，整体视觉保持简洁、清晰并适合大型工厂布局。

## 主要功能

- 中文工厂编辑器，支持画布平移、缩放、旋转、撤销和本地存档。
- 传送带支持直线、转弯、分流、合流、地下通道与跨线运输。
- 普通传送带与高速传送带拥有不同速度和连续插值动画。
- 铁矿、铜矿和煤矿可构成从金属锭、基础零件到处理器、伺服机构和自动化核心的 T0-T6 生产链。
- 熔炉拥有两个等价输入端，可烧制铁锭、铜锭与钢材。
- 合成器双击选择配方，能够生产高级零件与七类研究包。
- 研究中心四向接收研究包，双击选择项目后逐包消耗并解锁物流、配方和机器等级。
- 框选复制、相对蓝图粘贴、批量拆除和区域升级适用于大型工厂建设。
- Canvas 可视区域裁剪、模拟空间索引、自适应像素比和每帧补算限制用于减少卡顿。
- 底部状态栏显示 FPS、帧耗时、画质模式、交付、生产和堵塞情况。

## 生产链

基础工业线路：

    铁矿 + 煤矿 -> 熔炉 -> 铁锭
    铜矿 + 煤矿 -> 熔炉 -> 铜锭
    铁锭 -> 合成器 -> 铁板
    铜锭 -> 合成器 -> 铜线
    铁板 x2 -> 合成器 -> 齿轮
    铁板 + 铜线 x2 -> 合成器 -> 电路
    铁锭 + 煤矿 -> 熔炉 -> 钢材
    齿轮 + 电路 -> 合成器 -> 电机

完整线路、机器比例、研究节点和性能方案见 [生产与研究扩展设计](docs/production-research-expansion.md)。能源、流体化工、机器模块、高级物流、信号网络、列车、品质和终局目标的后续方案见 [综合扩展设计与实施路线](docs/future-expansion-master-plan.md)。

## 研究系统

研究不再由枢纽交付自动完成。玩家需要使用合成器生产物流、自动化、冶金、电子、机器人、核心和效用研究包，再通过传送带送入研究中心。

双击研究中心可选择七个项目：物流工程、自动化升级、冶金自动化、高级电子学、机器人技术、自动化核心和规模化生产。研究中心只消耗当前项目对应的研究包，前置项目未满足时不会吞料。

工业物品使用统一的层级图案：T0 圆形、T1 三角形、T2 正方形、T3 正五边形，后续层级依次增加边数。
## 大型建造

- 框选复制：拖出矩形，将区域保存为相对蓝图。
- 蓝图粘贴：点击格子作为蓝图左上角，保留方向、配方和机器等级。
- 批量拆除：一次清理矩形区域。
- 升级规划器：将普通传送带升级为高速传送带，并提升生产机器等级。
- 所有批量操作均可使用撤销按钮整步回退。

## 项目结构

    src/
      app/                     主应用与运行循环
      components/editor/       画布、状态栏和编辑器组件
      data/                    机器、物品、配方、研究和默认工厂
      engine/simulation/       固定 tick 模拟与编辑操作
      models/                  工厂数据模型
      render/                  Canvas 几何渲染

    tests/
      unit/                    单元测试
      integration/             模拟与生产闭环测试

    e2e/                       Playwright 浏览器测试
    docs/                      系统设计文档

## 性能设计

- 模拟使用固定 tick，Canvas 使用独立的 requestAnimationFrame。
- 每个 tick 创建一次实体空间索引，减少相邻格查询成本。
- 仅绘制当前视口及一格缓冲区内的实体。
- 低帧率时自动限制设备像素比，降低 Canvas 像素填充压力。
- 每帧最多补算 4 个模拟 tick，避免页面恢复焦点后连续卡顿。
- 普通和高速传送带按各自移动间隔计算动画进度，避免停顿和跳格。
