# FactoryGrid

FactoryGrid 是一个基于浏览器的模块化自动工厂模拟器，灵感来自异形工厂和异星工厂等自动化建造游戏。玩家可以在网格画布中放置机器、铺设传送带、设计生产路线，并通过运行模拟观察图形资源如何被加工、分流、合并、跨越和交付。

项目当前使用 Vue 3、TypeScript、Vite 与 Canvas 2D 实现。所有机器、传送带和物品都由简洁几何图形绘制，不依赖外部图片素材，适合继续扩展为完整的工厂沙盒、教学演示或自动化系统原型。

## 功能亮点

- 中文界面：工具栏、机器名称、运行状态和事件日志面向中文使用场景。
- 网格工厂画布：支持放置、删除、旋转、拖拽铺带、缩放和移动画布。
- 传送带系统：支持方向、转弯、预览、无缝连接和连续物品动画。
- 机器体系：包含矿脉、切割机、旋转器、染色器、堆叠器、分流器、合流器、地下通道、发射器、回收器和枢纽。
- 模拟引擎：固定 tick 推进生产逻辑，渲染层使用插值提升传送物品流畅度。
- 蓝图复用：可保存当前产线片段并复制到新区域，方便扩建大型工厂。
- 状态反馈：底部显示交付、生产、堵塞和事件日志，帮助定位产线问题。
- 本地保存：项目状态保存到浏览器本地存储，刷新后可以继续编辑。

## 快速开始

```bash
npm install
npm run dev
```

启动后在浏览器打开 Vite 输出的本地地址，例如：

```text
http://127.0.0.1:5173
```

指定端口启动：

```bash
npm run dev -- --port 5317
```

## 基本玩法

1. 从左侧工具栏选择矿脉、机器或传送带。
2. 在画布上点击放置建筑。
3. 拖拽铺设传送带，让物品沿指定方向流动。
4. 按 `R` 旋转当前选中机器或当前建造方向。
5. 点击运行，观察物品是否顺利进入机器和枢纽。
6. 根据底部状态栏调整堵塞、断线或方向错误的位置。
7. 保存蓝图，将稳定产线复制到更大的工厂布局中。

## 机器与组件

| 组件 | 作用 |
| --- | --- |
| 圆形/方形/星形/菱形矿脉 | 产生基础图形资源 |
| 传送带 | 按方向运输物品，支持转弯和连续动画 |
| 分流器 | 将输入物品交替分配到左右分支 |
| 合流器 | 接收多路输入并向一个方向输出 |
| 地下通道 | 跨越中间格子连接远端传送带 |
| 跨线发射器 | 将物品向前发射到远端位置 |
| 切割机 | 将图形切成半形 |
| 旋转器 | 旋转兼容图形 |
| 红色/蓝色/绿色染色器 | 将兼容图形染成目标颜色 |
| 堆叠器 | 将多个输入组合成复合图形 |
| 回收器 | 清理不需要的输入物品 |
| 枢纽 | 接收目标图形并推进交付进度 |

## 项目结构

```text
src/
  app/                     Vue 应用入口和主编辑器界面
  components/editor/       工厂画布、机器图标、模拟状态栏等组件
  data/                    机器、资源和示例工厂数据
  engine/simulation/       tick 模拟、放置、删除、旋转和铺带逻辑
  models/                  工厂项目、实体、资源和指标类型
  render/                  Canvas 渲染器与几何资产规划

tests/
  unit/                    单元测试
  integration/             模拟引擎集成测试

e2e/                       Playwright 浏览器测试
public/                    静态资源目录
```

## 架构概览

FactoryGrid 将编辑器、模拟引擎和渲染层拆开处理：

- 编辑器层负责用户输入，例如点击放置、拖拽铺带、移动画布、缩放、删除和旋转。
- 模拟层负责确定性 tick，处理资源生成、机器加工、传送带移动、地下通道、发射器、交付和堵塞检测。
- 渲染层只读取项目状态，用 Canvas 绘制网格、建筑、连接、预览和物品动画。

这种结构让生产逻辑更容易测试，也方便继续扩展更多机器、更多传送规则和更大的工厂规模。

## 测试覆盖

当前测试覆盖以下关键路径：

- 工厂可以运行并交付目标图形。
- 传送带拖拽预览、移动画布和蓝图保存可用。
- 机器不使用外部图片素材，全部由几何图形绘制。
- 传送带方向标识、转弯和连接逻辑保持一致。
- 地下通道可以成对连接并正确传递物品。
- 物品渲染使用固定步长插值，避免在格子中心跳动。
- 画布不再绘制遮挡视野的目标悬浮条。

完整验证命令：

```bash
npm run lint
npm test
npm run build
npm run test:e2e
```
