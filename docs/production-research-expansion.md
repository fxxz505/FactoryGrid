# FactoryGrid 多层生产、研究与大型建造系统设计

## 1. 文档目标

本文档定义 FactoryGrid 下一阶段的完整玩法闭环，覆盖以下四部分：

1. 从矿石到电机的多层工业生产链。
2. 由实际产品交付驱动的研究系统。
3. 面向大型工厂的框选、复制、蓝图粘贴、批量拆除与升级规划器。
4. 大规模传送带模拟、动画与帧率稳定方案。

设计继续遵循当前项目原则：中文界面、Canvas 2D、简单几何图形、无外部贴图、固定 tick 模拟、机器和传送带无缝连接。

## 2. 核心玩法闭环

矿物发生器
→ 熔炉与合成器
→ 铁板 / 铜线 / 齿轮 / 钢材 / 电路 / 电机
→ 枢纽交付
→ 获得研究点并累计研究产品
→ 解锁高速传送带 / 机器升级 / 高级配方
→ 使用框选与蓝图扩大产线
→ 提升吞吐量并完成更高阶研究

研究系统不使用脱离生产的货币按钮。玩家必须建立真实产线，把指定产品送进枢纽，研究才会推进。

## 4. 配方与基础节拍

基础模拟步长为 170ms。表中的 tick 是机器在 1 级状态下完成一次加工需要的逻辑 tick。

| 配方 | 机器 | 输入 | 输出 | 时间 |
| --- | --- | --- | --- | --- |
| 铁锭 | 熔炉 | 铁矿 ×1 + 煤矿 ×1 | 铁锭 ×1 | 34 tick |
| 铜锭 | 熔炉 | 铜矿 ×1 + 煤矿 ×1 | 铜锭 ×1 | 34 tick |
| 钢材 | 熔炉 | 铁锭 ×1 + 煤矿 ×1 | 钢材 ×1 | 52 tick |
| 铁板 | 合成器 | 铁锭 ×1 | 铁板 ×1 | 18 tick |
| 铜线 | 合成器 | 铜锭 ×1 | 铜线 ×2 | 22 tick |
| 齿轮 | 合成器 | 铁板 ×2 | 齿轮 ×1 | 28 tick |
| 电路 | 合成器 | 铁板 ×1 + 铜线 ×2 | 电路 ×1 | 42 tick |
| 电机 | 合成器 | 齿轮 ×1 + 电路 ×1 | 电机 ×1 | 56 tick |

为保持当前单物品实体模型，产量大于 1 的配方采用连续输出队列：铜线配方完成后生成两件铜线，输出端逐件放行。

## 5. 推荐完整线路

### 5.1 铁板基础线

铁矿发生器 → 传送带 → 熔炉 → 传送带 → 合成器（铁板）→ 铁板主总线。煤矿发生器从熔炉第二入口送入燃料。

推荐布局：

- y=0：铁矿源 → 传送带 → 熔炉 → 铁板合成器 → 铁板总线。
- y=1：熔炉燃料入口。
- y=2：煤矿源 → 传送带 → 向上转弯接入熔炉。

铁板是齿轮和电路共同使用的中间件，建议先进入一条横向主总线，再用分流器分成两支。

### 5.2 铜线基础线

铜矿发生器 → 传送带 → 熔炉 → 传送带 → 合成器（铜线）→ 铜线主总线。铁线和铜线可以共用煤矿主干，但必须使用分流器确保两个熔炉都能持续获得燃料。

### 5.3 齿轮支线

铁板主总线 → 分流器 → 合成器（齿轮）→ 齿轮缓冲线。

齿轮消耗 2 份铁板。建议齿轮线只占铁板总线的一条分支，避免电路生产完全断料。

### 5.4 电路支线

铁板主总线 → 分流器 → 合成器（电路）。
铜线主总线 → 分流器 → 同一合成器的另一输入端。
电路输出进入电路主总线。

合成器朝东时拥有西、北、南三个输入端。推荐西侧接铁板，北侧接铜线，南侧预留扩展或回流线路。逻辑上三个入口均按配方接收兼容物品，不要求固定材料侧。

### 5.5 钢材支线

铁锭出口 → 分流器 → 钢材熔炉。
煤矿主干 → 分流支线 → 钢材熔炉第二入口。
钢材输出进入研究交付线。

钢材会与铁板争夺铁锭。研究阶段建议将约三分之一铁锭送往钢材线，剩余铁锭继续生产铁板。

### 5.6 电机最终线

齿轮缓冲线 → 分流器 → 合成器（电机）。
电路主总线 → 分流器 → 同一合成器的另一输入端。
电机输出直接进入枢纽。

电机配方在“冶金自动化”研究完成后解锁。电机线应位于齿轮线和电路线汇合区域，避免两条长距离回传线路交叉。

### 5.7 推荐总线式工厂

- 铁矿 → 铁锭熔炉 → 铁锭总线 → 铁板 → 电路与齿轮。
- 铜矿 → 铜锭熔炉 → 铜线总线 → 电路。
- 煤矿 → 燃料总线 → 铁熔炉、铜熔炉、钢材熔炉。
- 铁锭分支 → 钢材熔炉 → 钢材研究线。
- 齿轮与电路汇合 → 电机合成器 → 枢纽。

总线之间至少保留一格空白，用于地下通道、分流器和后续升级。

## 6. 研究系统

### 6.1 研究点来源

枢纽接收工业产品时立即获得研究点，同时累计该产品的研究交付数量。

| 产品 | 每件研究点 |
| --- | --- |
| 铁板 | 1 |
| 铜线 | 1 |
| 齿轮 | 2 |
| 电路 | 4 |
| 钢材 | 5 |
| 电机 | 10 |

普通几何目标仍可计入原有交付目标，但不产生工业研究点。

### 6.2 研究节点

#### 节点一：物流工程

- 前置：无。
- 研究点：20。
- 产品条件：铁板 8、铜线 8。
- 解锁：高速传送带。
- 作用：普通传送带每 2 tick 移动一格，高速传送带每 1 tick 移动一格。

#### 节点二：自动化升级

- 前置：物流工程。
- 研究点：40。
- 产品条件：齿轮 8、电路 4。
- 解锁：升级规划器、机器 2 级。
- 作用：框选机器后统一升级；2 级机器加工时间缩短至基础时间的 75%。

#### 节点三：冶金自动化

- 前置：自动化升级。
- 研究点：70。
- 产品条件：钢材 8、电路 8。
- 解锁：电机配方。
- 作用：合成器双击配方面板出现电机配方。

#### 节点四：规模化生产

- 前置：冶金自动化。
- 研究点：100。
- 产品条件：电机 4、钢材 12。
- 解锁：机器 3 级。
- 作用：3 级机器加工时间缩短至基础时间的 55%。

### 6.3 自动推进规则

1. 研究节点严格按前置顺序检查。
2. 产品交付数量是累计值，不因研究完成而清零。
3. 研究点是可消费值，完成节点时扣除对应点数。
4. 同一 tick 内允许连续完成多个已经满足条件的节点。
5. 解锁结果写入存档，刷新页面后继续生效。

## 7. 大型工厂建造工具

### 7.1 框选复制

- 选择“框选复制”工具。
- 在画布上从任意角拖到另一角。
- 选区内机器保存为相对坐标，不保存输入、输出、加工进度和传送带物品。
- 完成框选后自动切换到“蓝图粘贴”。

### 7.2 蓝图粘贴

- 鼠标悬停时以半透明几何图形预览蓝图占用范围。
- 点击格子将蓝图左上角对齐该格。
- 与现有建筑重叠时采用覆盖规则，并清理被覆盖传送带的运行时物品。
- 配方、方向、机器等级和传送带类型会被保留。

### 7.3 批量拆除

- 选择“批量拆除”后拖出矩形区域。
- 一次删除区域内所有建筑和对应传送带运行时状态。
- 操作只记录一个撤销历史节点。

### 7.4 升级规划器

- 研究“自动化升级”后可用。
- 框选普通传送带时升级为高速传送带。
- 框选生产机器时提升一级，不超过研究允许的最大等级。
- 枢纽、回收器、发生器和纯物流路由器不升级。
- 整个区域升级只记录一个撤销历史节点。

### 7.5 蓝图数据结构

蓝图保存 type、相对 offset、direction、recipeId 和 level，而不是引用当前工厂实体 ID。旧存档中仅包含 entityIds 的蓝图在加载时仍可显示，但保存新蓝图时统一使用相对实体。

## 8. 传送带逻辑优化

### 8.1 速度分层

- 普通传送带：2 tick/格。
- 高速传送带：1 tick/格。
- 物品进入传送带时记录 enteredTick。
- 渲染进度使用“当前 tick - enteredTick + renderAlpha”除以移动间隔，保证不同速度都连续运动。

### 8.2 不丢件原则

1. 下一格被占用时，当前物品停在本格出口侧。
2. 目标机器拒收物品时，传送带不得清空自身物品。
3. 同一物品在一个 tick 内最多完成一次传送带移动。
4. 输出机器被堵塞时保留 output 队列，不重复消耗配方原料。

### 8.3 空间索引

模拟和渲染均使用“x,y → entity”的空间索引，避免每次查询都遍历全部实体。大型工厂中，相邻格查询从 O(n) 降为接近 O(1)。

## 9. 动画与帧率优化

### 9.1 固定模拟与独立渲染

- 模拟继续使用固定 tick，确保结果可重复。
- Canvas 使用 requestAnimationFrame。
- renderAlpha 只负责插值，不改变模拟状态。
- 每帧最多补算 4 个 tick，超过部分丢弃，避免页面恢复焦点后长时间卡死。

### 9.2 可视区域裁剪

仅绘制画布范围外一格缓冲区内的实体、物品与隧道连接。离屏工厂仍参与模拟，但不消耗 Canvas 绘制时间。

### 9.3 自适应画质

- FPS 不低于 50：清晰模式，最高 2 倍设备像素比。
- FPS 低于 50：流畅模式，最高 1.25 倍设备像素比并减少阴影细节。
- FPS 恢复稳定后自动返回清晰模式。
- 底部状态栏显示当前 FPS、帧耗时和画质模式。

### 9.4 帧率统计

使用 500ms 采样窗口和指数平滑，避免 FPS 文本每帧抖动。界面提示分为：

- 绿色：55 FPS 以上。
- 黄色：40 至 54 FPS。
- 红色：低于 40 FPS。

## 10. 模块闭环

| 模块 | 输入 | 输出 | 下游 |
| --- | --- | --- | --- |
| 资源模块 | 时间 tick | 矿石 | 熔炉 |
| 配方模块 | 原料、配方、机器等级 | 产品 | 物流与研究 |
| 物流模块 | 机器输出、传送带速度 | 到达目标的物品 | 机器或枢纽 |
| 研究模块 | 枢纽工业交付 | 研究点、解锁项 | 建造与配方面板 |
| 编辑器模块 | 框选区域、蓝图、升级权限 | 新布局 | 模拟与渲染 |
| 性能模块 | 实体规模、帧时间 | 画质等级、FPS | Canvas 与状态栏 |

## 11. 验收标准

- 铁矿和煤矿能经过完整传送带线路生产铁锭、铁板、齿轮、电路和电机。
- 铜矿和煤矿能生产铜锭与铜线，并参与电路配方。
- 铁锭和煤矿能生产钢材。
- 电机配方在对应研究完成前不可选择，完成后自动出现。
- 枢纽交付工业产品会增加研究点和研究交付计数。
- 研究完成后正确解锁高速传送带、升级规划器和高级配方。
- 框选复制、蓝图粘贴、批量拆除和升级区域均可撤销。
- 普通与高速传送带速度不同，且动画不跳格、不闪烁、不丢件。
- 大型工厂只绘制可见实体，FPS 状态提示持续更新。
- npm test、npm run build、npm run lint 和 npm run test:e2e 全部通过。
